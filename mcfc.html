McFc4 Documentation

Introduction

McFc4 (Minecraft Functions Version4) is a strictly typed Minecraft datapack. It is a coding language that compiles down into a datapack. McFc Version 4 is designed to run on Minecraft 1.19.2, but will most likely work on most other versions*

* Assuming that it doesn't modify any current commands or datapack structures/rules used by McFc.

History

McFc versions 2 and 3 are not available to the public due to them never being completed due to a high amount of bugs and a lack of proper maintainability in the code. All versions of McFc are written by Daniel Dobromylskyj and are all based on the same idea of a no-mod based coding language for Minecraft. As of version 4, none of the syntaxes are compatible with each other, due to significant changes in how it is compiled.

Extra Rules About McFc(4)

  • All If and Loop Statements must use the correct indentation of 4 spaces (Tab)
  • Comments are written using double forward slashes ("//"). They cannot come on the same line as code.
  • It is recommended to only run the mcfc:main function when attempting to run the program.

Variable Declaration

      
<Type> <Identifier> = <value>             Sets to <Value>
OR
<Type> <Identifier>                          Sets to 0 or 0.000

Examples:

      
int x = 5
float pie = 3.141

If Statements

      
if <var1> <operator> <var2>
OR
if not <var1> <operator> <var2>
<operator>: == > >= < <=
'not' inverts the output. True = False, False = True

Loops

Controlled Loop

      
LOOP <loopid>
    // loopstuff
NEXT <loopid>

The loopid must be all lowercase and not use any punctuation or spaces. Valid 'special' characters: _

The loop will only repeat when the NEXT keyword is called. Add an if statement to condition it.

ForAll Loops

  
ForAll <Type>

This loop doesn't need an ID and may not compile as intended if given. This loop goes through every entity/player in its range and runs the code as if the entity/player ran it themselves.

Valid <Type>(s):

  • players - Loop through code as every player (Currently Connected)
  • entities - Loop through code as every entity (Currently Loaded)

When using 'ForAll' Loops, the following variables are set according to the entity/player selected:

  • is_player: bool_int
  • Xpos: int
  • Ypos: int
  • Zpos: int

Functions

To make a function, create a new file, <function_name>.mcfunc. The <function_name> should be all lowercase and only use '_', no spaces or other characters.

      
#input <identifier1>:<type1>,<identifier2>:<type2>
#output <identifier3>:<type3>

Anything after #input will be an argument parsed to the function. As of 04/06/2023, there must only be one return argument and have no spaces between input arguments.

The identifier after #output will be returned. Outputs are not needed.

To call a function, use the following syntax:

  
    #import <function_name>
    CALL <function_name> <arg1>,<arg2> -> <returnValue>

You don't have to have a returnValue set to anything, just remove the "->" section. As of 04/06/2023, it is untested whether or not you can call a function with no arguments.

Execute Raw Minecraft Commands

      
RAW <Command>

Mc Commands (ForAll Use Only)

  • moveTo x y z: Teleport player/entity to this position. Relative values can also be used.
  • removeSelf: Kill current entity - Doesn't stop the loop.
